# 主页维护指南

这份站点是 Astro 项目。日常只需要记住三件事：内容放进仓库、图片也放进仓库、最后推送到 GitHub。

## 常用位置

- 首页内容：`src/pages/index.astro`
- 个人介绍页：`src/pages/about/index.astro`
- 站点标题、作者、菜单：`src/site.config.ts`
- 公开的日记文章：`src/content/blog/diary/`
- 原始日记备份：`Notes/Diary/`
- 网站图片：`public/uploads/`

## 写日记

网站真正读取的是 `src/content/blog/diary/` 里面的 Markdown。

每篇日记开头需要有 frontmatter，例如：

```md
---
title: "2026年7月7日"
description: "今天做了什么"
publishDate: 2026-07-07
tags: ['diary']
language: 'zh-CN'
comment: false
---

正文写在这里。
```

`Notes/Diary/` 只是保留原始日记的地方。为了避免忘记同步，最简单的办法是：需要发布到网页的日记，直接编辑 `src/content/blog/diary/` 里的版本。

## 上传图片

不要使用这种路径：

```md
![图](C:\Users\qrange\AppData\Roaming\Typora\typora-user-images\xxx.png)
```

这种路径只在你的电脑上存在，网页访问不到。

推荐做法：

1. 把图片放到 `public/uploads/diary/` 或 `public/uploads/其他分类/`。
2. 在 Markdown 里这样引用：

```md
![图片说明](/uploads/diary/image-20260707.png)
```

如果你用 Typora，可以把图片复制设置改成固定目录：

- 进入 Typora 设置里的 Image / 图片设置。
- 选择复制图片到自定义文件夹。
- 文件夹填：`C:\Users\qrange\Desktop\qrange-yzy.github.io\public\uploads\diary`
- 插入图片后，把 Markdown 里的链接改成 `/uploads/diary/文件名.png`。

## 写普通文章

普通博客文章放在 `src/content/blog/`。

如果文章只有一个文件，可以创建：

```text
src/content/blog/my-post.md
```

如果文章有很多图片，推荐创建一个文件夹：

```text
src/content/blog/my-post/index.md
src/content/blog/my-post/photo.png
```

然后在文章里用：

```md
![图片说明](./photo.png)
```

## 修改个人简介

首页简短介绍在 `src/pages/index.astro`。

更完整的 About 页面在 `src/pages/about/index.astro`。

站点名字、作者、菜单项在 `src/site.config.ts`。

## 发布流程

改完后执行：

```powershell
git status
git add -A
git commit -m "Update site content"
git push origin main
```

GitHub Actions 会自动构建并部署。通常 1 到 5 分钟后生效。如果页面没变，先强刷浏览器：`Ctrl + F5`。
